# 要件定義書：イベント参加者管理アプリ

## 1. 背景・目的

イベントの参加者を紙や表計算ソフトで管理していたが、
参加状況の確認や検索に時間がかかり、管理が煩雑だった。

そのため、参加者情報を登録・検索・管理できるアプリを作成する。

---

## 2. 利用者

イベントの参加者を管理したい個人または小規模なイベント運営者

---

## 3. 機能一覧

| 画面/機能    | 概要                               |
| -------- | -------------------------------- |
| 参加者一覧    | 名前・参加イベント・参加状況を一覧表示する            |
| 検索・フィルター | 名前で検索、参加状況（参加予定・参加済み・キャンセル）で絞り込み |
| 参加者登録    | 名前・メールアドレス・参加イベント・参加状況を入力して登録する  |
| 参加状況変更   | 登録済み参加者の参加状況を変更できる               |
| 参加者削除    | 登録した参加者を削除する                     |

---

## 4. 非機能要件

* （要検討：参加者は何人まで登録できるようにするか）
* （要検討：メールアドレスの重複登録を許可するか）
* （要検討：イベント名は自由入力にするか、選択式にするか）

---

## 5. 制約条件・業務ルール（意図的に曖昧にしている箇所）

* 名前は必須入力とする。
* メールアドレスは正しい形式で入力すること。
* 参加状況は「参加予定」「参加済み」「キャンセル」のいずれかを選択する。
* （要検討：同じイベントへ同じメールアドレスで複数回登録を許可するか）
* （要検討：参加人数の上限を設定するか）
